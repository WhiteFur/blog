Using Chakram to test your Restful API

2015-11-06
#Chakram #BDD #Testing

Chakram是一套針對Restful API的testing framewok
其建構在Mocha, Chai之上
它利用ECMAScript 6的Promise讓programmer在測試非同步的行為時，能夠寫出更易懂的測試程式碼

注意：

  1. 本文預設讀者已經瞭解Mocha的基本運作流程，所以不針對Mocha做介紹
  2. 本文僅粗略的介紹大概的寫法，請搭配Chakram的文件

以下為一測試post的例子

  • post url: http://localhost:8080/ApServer/Slot/CheckCashierActiveSession
  • arguments: workstationID: 20008, categoryIdx:11
  1. checkram.post returns a Promise object, which will resolve to a ChakramResponse object
  2. 用Expect對上述promise object做測試
  3. chakram.wait() 會return一個promise, 該promise會等待與其同一層的所有chakram expectations都被fufilled時, 才會跟著fullfill
  4. Mocha的it如果return一個promise, 則Mocha會等該promise被fulfill, 才會完成此it block的測試

再來看看稍微複雜一點的例子

當我們必須使用response的值來做為另一支api的參數時，可利用promise的特性，用.then來串接下去

測試寫完後，在terminal下mocha指令，開始跑測試

mocha預設會去執行./test/ 資料夾下的所有js file，也可以指定要跑的測試檔
