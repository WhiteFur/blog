Capital

2015-12-18
#Capital #Vault
  • Fill - 金庫送錢給Workstation
  • Credit - Workstation送錢給金庫

DB Table

  1. CapitalRecord: 每次發起fill/credit會寫一筆進來 (只記錄總額)
    • CategoryIdx: 目標Workstation的CategoryIdx, 用來辨別該Workstation是屬於slot or table
    • Type: 0 is Cash, 1 is Chip (for Table System)
  2. CapitalDetailRecord: CapitalRecord的各面額數量, 所以一筆CapitalRecord有好幾筆CapitalDetailRecord
  3. CapitalRecordTransList: 表示CapitalRecord的Status轉換(每改變一次就會寫一筆), 所以一筆CapitalRecord有好幾筆CapitalDetailRecord
  4. CapitalRecordStatus: CapitalRecord的Status定義

運作流程

  1. REQUEST: 由Cashier發起request
  2. VALIDATING: Vault收到後, 並且將該request點開
  3. SENDING: Vault點完鈔後, 按下confirm, 實際把錢運送出去
  4. RETURNING: Cashier收到錢, 因故拒收(e.g.金額不正確), 把錢退回
  5. COMPLETED: Cashier or Vault收到錢確認無誤, 點收完畢
  6. ABORT: Cashier發出request後, 在Vault validate前, 取消該request
  7. REJECT: Vault點開request後, 拒絕該request
  8. RETURN RECEIVED: Vault收到被退回的錢
  9. TO PICK UP: credit時, 等待vault派人來cashier收錢

以下三者為table system的賭桌間交換chip時專用
