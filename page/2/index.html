file_encoding

Posted on 2016-02-23

如何把file轉成我們想要的encoding

分為兩個部分

  • Determine the encoding
  • Convert to another encoding

Determine the encoding

有兩種做法

  1. 使用Unix的file command

    不過有時候會猜不準
    e.g. app.js明明是utf-8，但是因為都是英數字元，所以「file」就以為app.js是ASCII

  2. 使用vim的fileencdoing

    得知該檔的encoding

Convert to another encoding

有兩種做法

  1. 使用Unix的iconv

    -f encoding, –from-code=encoding
    Specifies the encoding of the input.

    -t encoding, –to-code=encoding
    Specifies the encoding of the output.

    list the supported encodings

  2. 用vim指定編碼再存檔

事實上，
我們無從得知file encoding為何，
VIM也只是根據檔案內容去猜它的encoding而已，
除非該檔有寫bom明確指出該檔案的encoding

There really isn’t a common way to determine the encoding of a plaintext file, as that information isn’t saved in the file itself - except UTF-8 Files where you’ve got a so called BOM which indicates the Encoding. This is why xml and html files have charset metatags.

Note that files’ encoding is not explicitly stated anywhere in a file. Thus, VIM and other applications must guess at the encoding.

詳細做法如下：

  1. 自定table type

  2. store procedure的parameter便可以使用此type(規定必須設為READONLY)

  3. execute時就能傳入一個該type的參數

Capital

Posted on 2015-12-18
  • Fill - 金庫送錢給Workstation
  • Credit - Workstation送錢給金庫

DB Table

  1. CapitalRecord: 每次發起fill/credit會寫一筆進來 (只記錄總額)
    • CategoryIdx: 目標Workstation的CategoryIdx, 用來辨別該Workstation是屬於slot or table
    • Type: 0 is Cash, 1 is Chip (for Table System)
  2. CapitalDetailRecord: CapitalRecord的各面額數量, 所以一筆CapitalRecord有好幾筆CapitalDetailRecord
  3. CapitalRecordTransList: 表示CapitalRecord的Status轉換(每改變一次就會寫一筆), 所以一筆CapitalRecord有好幾筆CapitalDetailRecord
  4. CapitalRecordStatus: CapitalRecord的Status定義

運作流程

  1. REQUEST: 由Cashier發起request
  2. VALIDATING: Vault收到後, 並且將該request點開
  3. SENDING: Vault點完鈔後, 按下confirm, 實際把錢運送出去
  4. RETURNING: Cashier收到錢, 因故拒收(e.g.金額不正確), 把錢退回
  5. COMPLETED: Cashier or Vault收到錢確認無誤, 點收完畢
  6. ABORT: Cashier發出request後, 在Vault validate前, 取消該request
  7. REJECT: Vault點開request後, 拒絕該request
  8. RETURN RECEIVED: Vault收到被退回的錢
  9. TO PICK UP: credit時, 等待vault派人來cashier收錢

以下三者為table system的賭桌間交換chip時專用
A2B SENDING
B2A SENDING
EXCHANGE SENDING
