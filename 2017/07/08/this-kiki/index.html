2017-07-08
#javascript #this #js #nodejs #kiki

比較需要詳細探討的應該是第14行的fOut()
this在simple function call (直接呼叫function的意思)時會指向global object

global object

  • browser - window
  • Nodejs - global

在browser中，宣告的變數或是function會綁定到window的property
所以你打開chrome的console試試看，window.x跟window.fOut都是有東西的

而在nodejs中，一支js file為一個module，module其實最後會被wrap成一個function來執行
事實上在nodejs最後執行的時候會是像這樣

x跟global當然就不會有任何關係
所以才會像你看到的
fOut()的結果是undefined

你可以做個實驗
把x綁定到global就可以印出10了

另外還有件值得一提的, 直接使用this跟在simple function call裡的this是不一樣的

所以可以在nodejs中做個實驗
