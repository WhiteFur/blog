Git-case-sensitivity

Posted on 2016-08-22

checkout時大小寫不影響

e.g., 目前remote repo有兩個branch

checkout時故意把TM打成Tm

仍然可以正常checkout

push時大小寫則會影響

我們會發現它track的remote branch的name大小寫也會跟local branch的一樣

做了一些修改, 再push

會發生error

因為git push origin feature/Tm-1的意思是把local的feature/Tm-1給push上去，如果沒有指定remote branch，git會認為是要push到origin/feature/Tm-1，但是origin remote並沒有feature/Tm-1，而是feature/TM-1

現在我們改成原來的大寫

結果找不到這個branch，因為local branch name為feature/Tm-1

必需指定要push到哪個remote branch才會成功

Conclusion

大小寫都一致就不會發生以上情形
