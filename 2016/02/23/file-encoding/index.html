如何把file轉成我們想要的encoding

分為兩個部分

  • Determine the encoding
  • Convert to another encoding

Determine the encoding

有兩種做法

  1. 使用Unix的file command

    不過有時候會猜不準
    e.g. app.js明明是utf-8，但是因為都是英數字元，所以「file」就以為app.js是ASCII

  2. 使用vim的fileencdoing

    得知該檔的encoding

Convert to another encoding

有兩種做法

  1. 使用Unix的iconv

    -f encoding, –from-code=encoding
    Specifies the encoding of the input.

    -t encoding, –to-code=encoding
    Specifies the encoding of the output.

    list the supported encodings

  2. 用vim指定編碼再存檔

事實上，
我們無從得知file encoding為何，
VIM也只是根據檔案內容去猜它的encoding而已，
除非該檔有寫bom明確指出該檔案的encoding

There really isn’t a common way to determine the encoding of a plaintext file, as that information isn’t saved in the file itself - except UTF-8 Files where you’ve got a so called BOM which indicates the Encoding. This is why xml and html files have charset metatags.

Note that files’ encoding is not explicitly stated anywhere in a file. Thus, VIM and other applications must guess at the encoding.
