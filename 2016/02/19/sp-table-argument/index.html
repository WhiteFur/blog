Use Table-Valued Parameters

Veröffentlicht am 2016-02-19

由於SQL沒有Array的型態可用
那要傳入多筆資料做為store procedure的參數時該怎麼做呢？
我們可以把欲傳入store procedure的資料insert到一個table variable
執行store procedure時再把該table variable當做參數傳入
詳細做法如下：

  1. 自定table type

  2. store procedure的parameter便可以使用此type(規定必須設為READONLY)

    Table-valued parameters must be passed as input READONLY parameters to Transact-SQL routines. You cannot perform DML operations such as UPDATE, DELETE, or INSERT on a table-valued parameter in the body of a routine.

  3. execute時就能傳入一個該type的參數

欲瞭解更多細節可以參考MSDN的說明
https://msdn.microsoft.com/en-us/library/bb510489.aspx
